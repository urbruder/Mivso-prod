# Buyer Pre-Approval Verifier — Reference

## Use This Reference When Running the Skill
Read this file before creating the deliverable. Use it as a compact checklist and language library, not as text to copy word-for-word.

## Pre-Approval Review Checklist
| Item | Why It Matters |
|---|---|
| Lender name and contact | Allows confirmation with buyer permission. |
| Date and expiration | Prevents relying on stale approval. |
| Loan amount | Helps compare approval to target price. |
| Loan type | Affects property type, condition, and offer strength. |
| Property type limits | Important for condos, manufactured homes, multi-unit, or new construction. |
| Conditions | Shows what still needs to be completed. |
| Down payment or cash-to-close if provided | Helps flag offer feasibility without giving loan advice. |

## Risk Flags
Expired letter, prequalification instead of pre-approval, missing lender, approval below target price, unclear property type, major unresolved conditions, no permission to verify, or buyer unwilling to discuss financing readiness.

## Readiness Summary
Ready means key details are present and timeline matches. Needs Clarification means one or more important details are missing. Not Ready means the buyer lacks verification or has a major mismatch with the intended showing or offer.


## Final Quality Check
- The deliverable is a finished visual file, not a raw chat response.
- The client can send, print, or present it without another formatting step.
- The language sounds like a helpful real estate professional, not a generic marketer.
- The file avoids protected-class language, unsupported claims, pressure tactics, and promises of results.
- Every recommended next step is practical, dated where useful, and tied to the lead's situation.
