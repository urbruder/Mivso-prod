# Drip Campaign Personalizer — Reference

## Use This Reference When Running the Skill
Read this file before creating the deliverable. Use it as a compact checklist and language library, not as text to copy word-for-word.

## Segmentation Matrix
| Dimension | Examples | How It Changes the Message |
|---|---|---|
| Persona | Buyer, seller, investor, past client | Changes education topic and CTA. |
| Source | Portal, open house, referral, ad, sphere | Changes opening context. |
| Stage | New, warm, nurture, active, stalled | Changes urgency and follow-up timing. |
| Timeline | Now, 30 days, 6 months, 12 months | Changes ask strength. |
| Behavior | Viewed property, replied, clicked, attended | Changes personalization line. |

## Personalization Rules
- Personalize the first sentence, not every sentence.
- Reference a real action or stated preference.
- Keep one CTA per message.
- Avoid fake intimacy when the relationship is cold.
- Create a fallback version when a personalization field is missing.

## QA Red Flags
Too many exclamation points, vague market claims, no local context, no clear next step, unsupported urgency, no opt-out path, protected-class language, or copy that could apply to anyone.


## Final Quality Check
- The deliverable is a finished visual file, not a raw chat response.
- The client can send, print, or present it without another formatting step.
- The language sounds like a helpful real estate professional, not a generic marketer.
- The file avoids protected-class language, unsupported claims, pressure tactics, and promises of results.
- Every recommended next step is practical, dated where useful, and tied to the lead's situation.
