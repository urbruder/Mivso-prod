# Open House Sign-In Optimizer — Reference

## Use This Reference When Running the Skill
Read this file before creating the deliverable. Use it as a compact checklist and language library, not as text to copy word-for-word.

## Sign-In Question Hierarchy
Required core fields should be minimal: name, phone or email, reason for visit, and follow-up permission. Add qualifying questions only when they help the visitor receive something useful.

## High-Value Questions
- Are you currently working with an agent?
- Are you exploring this home, similar homes, or the neighborhood market?
- What is your ideal timeline?
- Would you like the disclosure package, price updates, or similar listings?
- Are you a neighbor interested in how this open house performs?

## Visitor Value Offers
Disclosure package, similar homes, price-change alerts, offer-deadline updates, neighborhood market recap, home value review, or private showing times.

## Door Greeting
“Welcome in. We are asking visitors to sign in so I can send the property details and any updates after today. Are text or email updates easier for you?”


## Final Quality Check
- The deliverable is a finished visual file, not a raw chat response.
- The client can send, print, or present it without another formatting step.
- The language sounds like a helpful real estate professional, not a generic marketer.
- The file avoids protected-class language, unsupported claims, pressure tactics, and promises of results.
- Every recommended next step is practical, dated where useful, and tied to the lead's situation.
