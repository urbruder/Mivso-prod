# Sphere Reactivation Writer — Reference

## Use This Reference When Running the Skill
Read this file before creating the deliverable. Use it as a compact checklist and language library, not as text to copy word-for-word.

## Segment-to-Message Map
| Segment | Best Angle | CTA |
|---|---|---|
| Past client | Appreciation and home anniversary | “Want a quick equity update?” |
| Old buyer lead | Life update and search reset | “Are you still thinking about a move?” |
| Old seller lead | Market check-in | “Would updated numbers be helpful?” |
| Vendor | Relationship and mutual referrals | “Who should I know right now?” |
| Friend or neighbor | Personal check-in | “How have you been?” |
| Unknown cold contact | Permission-based restart | “Should I keep you on my local update list?” |

## Message Rules
- Keep texts under 320 characters unless the agent asks for longer.
- Mention one real connection point when available.
- Ask one clear question.
- Avoid multiple asks in the same message.
- Make it easy for the contact to say no.

## Follow-Up Timing
- First touch: day 1.
- If reply: respond the same day with a helpful next step.
- No reply: light second touch after 5–7 days.
- Still no reply: close-the-loop after 14–21 days.


## Final Quality Check
- The deliverable is a finished visual file, not a raw chat response.
- The client can send, print, or present it without another formatting step.
- The language sounds like a helpful real estate professional, not a generic marketer.
- The file avoids protected-class language, unsupported claims, pressure tactics, and promises of results.
- Every recommended next step is practical, dated where useful, and tied to the lead's situation.
