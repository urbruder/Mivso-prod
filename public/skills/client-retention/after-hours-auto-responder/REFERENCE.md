## Output Quality Standard
- Produce the visual deliverable named in SKILL.md, not a raw chat answer.
- Make the first page, first sheet, first slide, or top visual section useful as an executive summary.
- Put recommendations, risks, owners, dates, and next actions where the agent can find them quickly.
- Use plain real estate language and avoid generic filler.
- Make every table, checklist, slide, or visual block usable by a busy real estate professional.
- Add a Verify Before Sending section for assumptions, dates, numbers, compliance-sensitive wording, third-party facts, and legal or tax boundaries.

## Compliance Reminders
Use only personalization the agent has permission to use. Do not reference protected-class details, private family information, medical facts, or financial assumptions. Respect unsubscribe, do-not-contact, privacy, and brokerage communication policies.

## Runtime Clarifying Questions
Ask only the questions needed to complete the file. Prioritize these questions:
1. Who is the audience for the final deliverable?
2. What decision, action, or message should the deliverable support?
3. What facts, numbers, dates, or documents are already known?
4. What local, brokerage, lender, MLS, title, lease, owner, or legal rule did the agent supply?
5. What tone should the final client-facing language use?

## Final File Checklist
- The file type matches the Output Format in SKILL.md.
- The content is organized with clear headings, tables, visuals, or sections.
- The deliverable uses only facts supplied by the agent or clearly marked assumptions.
- Protected-class, steering, privacy, lending, legal, tax, title, lease, and advertising risks are handled carefully.
- The final section tells the agent what to verify before use.
